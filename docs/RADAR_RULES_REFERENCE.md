## 2. 雷达硬件制作规范

> 来源：③机器人制作规范手册 V1.3.0

### 2.1 整体结构

**S127**：雷达由**运算平台端**与**传感器端**构成，两者需通过电缆连接。

### 2.2 运算平台端

**S128**：

| 参数 | 限制 |
|------|------|
| 最大功率 | 750W |
| 供电 | 220V 市电 |
| 最大尺寸 | 600mm × 350mm × 600mm（长×宽×高） |

**S130**：运算平台端**不得安装或使用无线设备**。

### 2.3 传感器端

**S129**：

| 参数 | 限制 |
|------|------|
| 长宽尺寸 | 不得超出放置平台 |
| 高度 | ≤ 1500mm |
| 供电电压 | ≤ 60V |

### 2.4 安装规范

**S131**：传感器端需安装在传感器支架上。传感器支架由组委会提供安装于雷达基座上。

**S132**：激光发射装置需安装在传感器端上，并且激光发射装置及其执行机构需由裁判系统电源管理模块 Gimbal 接口供电。

**S133**：传感器端安装后，其整体（含支架）高度不得超过围栏高度，以避免影响比赛。

**S134**：运算平台端安装于雷达基座平台上，需放置稳固。

**S135**：雷达传感器端与运算平台端之间的连接电缆需走线合理，不得影响场地其他设施。

### 2.5 无线设备限制

**S21**：机器人不得搭载除遥控器、裁判系统、无线充电装置、**雷达信息接收装置**以外的无线通讯设备。

> **注意**：此条说明其他机器人可以搭载"雷达信息接收装置"，这是多机通信的硬件基础。

**S130**（重申）：雷达运算平台端不得安装或使用无线设备。即雷达的多机通信通过裁判系统有线链路实现，而非无线。

## 4. 裁判系统配置

> 来源：③机器人制作规范手册 裁判系统配置表 / ②裁判系统用户手册

### 4.1 雷达裁判系统配置

| 模块 | 数量 | 说明 |
|------|------|------|
| 主控模块 | 1 | 组委会安装于场地，雷达通过有线接入方式使用 |
| 电源管理模块 | 1 | Gimbal 接口连接激光发射装置及其执行机构电源 |

**S140 / S142**：雷达需使用组委会安装于场地的主控模块（有线接入方式）。雷达不自带主控模块，而是使用场地预装的。

### 4.2 裁判系统通信相关

> 来源：②裁判系统用户手册

裁判系统为雷达提供以下通信能力：

- **多机通信（机器人间交互数据）**：雷达可通过裁判系统向所有己方机器人发送数据
- **数据接收**：雷达仅能接收来自**哨兵机器人**的数据
- **哨兵通信对称性**：哨兵可以向所有己方机器人发送数据，但只能接收**雷达**的数据

裁判系统用户手册中定义了机器人间交互数据的协议格式，雷达可通过串口与主控模块通信，发送/接收多机通信数据包。

## 5. 雷达基座与场地设施

> 来源：④比赛规则手册 第4.2节 / 第4.2.5节

### 5.1 雷达基座

雷达基座位于**基地区**内，是雷达的专用安装位置。

| 参数 | 规格 |
|------|------|
| 平台高度 | 2.5m |
| 平台面积 | 3.4m × 1.16m |
| 围栏高度 | 1.1m |
| 配备设施 | HDMI 显示器 + 信号线 + 五孔插座 |
| 供电 | 220V 市电 |

**详细说明**：

- 雷达基座是一个高 2.5m 的平台，平台面积为 3.4m × 1.16m
- 平台四周设有高度为 1.1m 的围栏
- 平台上配备 HDMI 显示器及信号线、五孔插座，供雷达运算平台端使用
- 雷达运算平台端放置于平台上，传感器端安装于组委会提供的传感器支架上
- 供电为 220V 市电，通过五孔插座提供

### 5.2 操作间相关

> 来源：④比赛规则手册 操作间相关规定

飞手操作间仅配备**护目镜**，用于防止**对方雷达激光照射**。

> **说明**：这表明雷达激光虽然功率≤1mW，但仍需对飞手提供视觉保护，操作间配备护目镜是标准安全措施。

## 6. 雷达标记机制（被标记进度）

> 来源：④比赛规则手册 第5.6.6节

### 6.1 雷达识别机制概述

雷达可以对敌方机器人进行**位置识别**。当雷达将识别到的敌方机器人坐标发送给裁判系统后，裁判系统会判定该坐标与敌方机器人实际位置的偏差：

- 若偏差在**允许范围内**，则判定为**"准确标记"**
- 若偏差超出允许范围，则判定为**"错误标记"**

每台敌方地面机器人都有一个**"被标记进度"** P 值，初始为 0。

### 6.2 被标记进度 P 值计算

**P 值范围**：0 ≤ P ≤ 150

**P 值变化规则**：

| 条件 | P 值变化 | 说明 |
|------|---------|------|
| 准确标记 | 每秒 +5 | 雷达坐标与实际位置偏差在允许范围内 |
| 错误标记 | 每秒 -10 | 雷达坐标与实际位置偏差超出允许范围 |
| 未被标记 | 每秒 -10 | 雷达未发送该机器人坐标 |

**关键说明**：
- P 值在 0 和 150 之间钳制（不会低于 0 或高于 150）
- 同一时刻，雷达可以同时标记多个敌方机器人
- P 值是**逐机器人独立计算**的，每台敌方地面机器人有各自的 P 值

### 6.3 被标记阈值与易伤效果

当某台敌方机器人的 P 值达到特定阈值时，会触发**易伤效果**（受到额外伤害）：

| P 值阈值 | 效果 |
|----------|------|
| P ≥ 100 | 该机器人进入**易伤状态**，受到伤害时有额外伤害增益 |

**易伤效果详细说明**：

- 当 P ≥ 100 时，该敌方机器人受到己方机器人攻击时，伤害会获得**额外增益**
- 易伤状态会持续到 P 值降回 100 以下
- 易伤增益与具体伤害计算公式详见[第6.5节](#65-增益计算示例)

### 6.4 双倍易伤机制

> 来源：④比赛规则手册 第5.6.6节

**触发条件**：当雷达成功解析电磁波（详见[第8节](#8-雷达解析电磁波机制)）后，可获得**双倍易伤**效果。

**双倍易伤规则**：

| 参数 | 值 |
|------|-----|
| 每场比赛最多触发次数 | **2次** |
| 每次持续时间 | **30秒** |
| 效果 | 被标记机器人的易伤伤害增益**翻倍** |

**详细说明**：
- 双倍易伤是全局效果，在持续时间内所有满足 P ≥ 100 的敌方机器人均受到双倍易伤增益
- 每场比赛最多触发2次双倍易伤，用完后本场不再可用
- 双倍易伤需要通过雷达成功解析电磁波（信息波）来激活

### 6.5 增益计算示例

> 来源：④比赛规则手册 增益计算示例

伤害计算中涉及雷达标记进度的增益示例：

- 当敌方机器人被雷达标记且 P ≥ 100 时，该机器人受到攻击的伤害会乘以一个**增益系数**
- 在**双倍易伤**期间，该增益系数翻倍
- 具体增益数值由裁判系统根据当前比赛状态实时计算

> **开发提示**：雷达的核心价值在于持续、准确地标记敌方机器人，使其 P 值维持在 100 以上，从而为己方攻击提供伤害增益。标记精度和持续性是雷达性能的关键指标。

## 7. 雷达反制空中机器人

> 来源：④比赛规则手册 第5.6.3节

### 7.1 反制机制概述

雷达可通过**激光发射装置**照射对方**空中机器人（飞行机器人）**上的**激光检测模块**，实现对空中机器人的反制。

反制成功后，空中机器人将受到惩罚效果（如强制降落或限制飞行能力）。

### 7.2 激光照射与被反制进度

空中机器人有一个**"被反制进度"** P 值，范围为 **0 ≤ P ≤ 100**。

| 条件 | P 值变化 | 说明 |
|------|---------|------|
| 激光持续照射到激光检测模块 | 每秒增加 | 雷达激光照射在空中机器人的激光检测模块有效区域内 |
| 激光未照射 | 每秒减少 | 雷达激光未照射到激光检测模块 |

**被反制触发**：当 P 值达到 **100** 时，空中机器人被反制成功。

### 7.3 激光检测模块面积递减

> 来源：④比赛规则手册 第5.6.3节

激光检测模块的**有效检测面积**会随着空中机器人被成功反制的次数**逐步递减**，即：

- 第一次反制：激光检测模块的有效检测区域为初始大小
- 后续每次成功反制后，有效检测区域面积减小
- 这意味着**反制难度逐步增加**——随着反制次数增多，雷达需要更精准地照射才能命中缩小的检测区域

> **开发提示**：这一递减机制要求雷达的激光瞄准精度需要非常高，且应具备跟踪空中机器人运动的能力。云台的伺服控制精度是关键。

### 7.4 被反制效果

空中机器人被雷达反制成功后：

- 空中机器人将受到**惩罚效果**（如功能受限或强制降落等裁判系统自动触发的效果）
- 反制成功的计数会影响后续激光检测模块的有效面积（递减机制）
- 反制次数同时计入雷达评分（详见[第12节](#12-雷达竞技评分标准)）

## 8. 雷达解析电磁波机制

> 来源：④比赛规则手册 第5.6.6节

### 8.1 机制概述

比赛场地中会持续发射**电磁波信号**，雷达可通过射频接收设备（如 PlutoSDR）接收并解析这些信号。电磁波分为两种类型：

| 类型 | 说明 |
|------|------|
| **信息波** | 包含有效密钥数据，成功解析后可激活双倍易伤 |
| **干扰波** | 干扰信号，不包含有效数据，需要区分排除 |

### 8.2 信息波与干扰波

**核心挑战**：雷达需要从混合的电磁波信号中**区分信息波和干扰波**，并成功解调信息波中的密钥。

- 信息波和干扰波在频谱上有不同的特征分布
- 信息波采用 **2-GFSK** 调制方式
- 干扰波可能在频谱上与信息波重叠，增加区分难度

### 8.3 难度等级与密钥

信息波的解析分为不同的**难度等级**：

| 参数 | 说明 |
|------|------|
| 密钥长度 | **6个字符** |
| 难度等级 | 多个等级，难度递增 |
| 解析方式 | 射频接收 → 2-GFSK 解调 → 提取密钥 → 提交裁判系统验证 |

**难度递增机制**：

- 更高难度等级的信息波可能具有更低的信噪比、更多的干扰波混合、或更复杂的调制参数
- 成功解析更高难度等级的信息波，在评分中获得更高分数

### 8.4 解析成功效果

当雷达成功解析信息波并将密钥提交给裁判系统验证通过后：

1. **激活双倍易伤**：己方雷达标记的所有满足条件（P ≥ 100）的敌方机器人，易伤增益翻倍
2. **持续时间**：双倍易伤持续 **30秒**
3. **次数限制**：每场比赛最多激活 **2次** 双倍易伤
4. **评分加成**：成功解析的难度等级计入雷达竞技评分

> **开发提示**：电磁波解析是雷达的高级功能，需要软件定义无线电（SDR）技术。推荐使用 PlutoSDR 等设备在 433MHz 频段接收信号，进行 2-GFSK 解调和帧解析。

## 9. 雷达信息波特性（附录）

> 来源：④比赛规则手册 附录1

### 9.1 频谱分布

信息波的频谱特性：

- 工作频段：**433MHz 频段**
- 信息波和干扰波在频谱上有不同的分布特征
- 雷达需要通过频谱分析区分信息波与干扰波

### 9.2 空口帧结构

信息波的空口帧（Air Interface Frame）结构如下：

```
┌──────────────┬──────────────┬──────────────┐
│  Access Code │    Header    │   Payload    │
│  （接入码）   │   （帧头）    │  （有效载荷） │
└──────────────┴──────────────┴──────────────┘
```

| 字段 | 说明 |
|------|------|
| **Access Code**（接入码） | 帧同步和接入标识，用于帧检测和同步 |
| **Header**（帧头） | 包含帧控制信息 |
| **Payload**（有效载荷） | 包含实际的密钥数据 |

### 9.3 2-GFSK 时域特征

信息波使用 **2-GFSK（2-Gaussian Frequency Shift Keying）** 调制方式：

| 参数 | 说明 |
|------|------|
| 调制方式 | 2-GFSK |
| 含义 | 高斯滤波的二元频移键控 |
| 特点 | 两个频率代表二进制 0 和 1，频率转换经过高斯滤波平滑 |

2-GFSK 的时域特征：
- 信号频率在两个值之间切换，代表二进制数据
- 高斯滤波使频率转换更加平滑，减少带外辐射
- 解调时需要进行频率鉴别，恢复二进制数据流

### 9.4 数据帧协议

信息波的数据帧协议定义了密钥数据的封装格式：

- 有效载荷中包含 **6字符密钥**
- 数据帧遵循特定的协议格式，需要按照协议解析出密钥
- 解析出的密钥通过裁判系统接口提交验证

> **开发提示**：完整的解析流程为：SDR设备接收433MHz信号 → 频谱分析区分信息波/干扰波 → 2-GFSK解调 → 帧同步（Access Code检测） → 帧头解析 → 有效载荷提取 → 密钥解码 → 通过裁判系统串口提交密钥。

## 10. 比赛流程中的雷达规则

> 来源：④比赛规则手册

### 10.1 准备阶段

- 雷达需在准备阶段**提前布置**，安装于**雷达基座**上
- 传感器端安装在组委会提供的传感器支架上
- 运算平台端放置于雷达基座平台上
- **R22**：三分钟准备阶段结束后，雷达的遥控器需放置在**指定区域**，不得继续手持操作

> **说明**：雷达在准备阶段可以使用遥控器进行调试（如调整传感器角度），但比赛正式开始后雷达必须完全自动运行。

### 10.2 比赛阶段

- 雷达在比赛中为**自动运行**模式，不需要也不允许操作手实时控制
- 最多允许 **1个遥控器** 用于调试目的（准备阶段使用，比赛开始后需放下）
- 雷达不参与场上移动和攻击，固定安装在雷达基座上
- **R56**：不可主动攻击对方雷达（雷达受规则保护）

### 10.3 多机通信

> 来源：④比赛规则手册 第3.7节 / 哨兵相关规定

**雷达的多机通信规则**：

| 方向 | 规则 |
|------|------|
| **发送** | 雷达可向**所有己方机器人**发送数据 |
| **接收** | 雷达**仅能接收哨兵机器人**的数据 |

**哨兵的多机通信规则**（对称关系）：

| 方向 | 规则 |
|------|------|
| **发送** | 哨兵可向**所有己方机器人**发送数据 |
| **接收** | 哨兵**仅能接收雷达**的数据 |

**通信拓扑说明**：
- 雷达和哨兵形成了一个特殊的**双向通信对**
- 雷达可以广播给所有己方机器人（包括哨兵），但只接收哨兵的回传
- 其他机器人可以接收雷达的数据，但无法向雷达发送数据
- 多机通信通过裁判系统有线链路实现

## 11. 异常情况与处罚

> 来源：④比赛规则手册 第5.1.5节 / 犯规条款

### 11.1 雷达异常离线

> 来源：④比赛规则手册 第5.1.5.1节

当雷达异常离线时，裁判系统将自动执行以下措施：

- **多机通信断开**：雷达无法再向己方机器人发送数据
- **激光发射装置及其执行机构断电**：激光反制功能失效

### 11.2 定位模块离线

> 来源：④比赛规则手册 第5.1.5.4节

当某台机器人的**定位模块离线**时，与雷达标记相关的判定规则如下：

| 离线方 | 效果 |
|--------|------|
| **对方机器人**定位模块离线 | 对方雷达对该机器人的标记**始终视为"准确"** |
| **己方机器人**定位模块离线 | 己方雷达对该机器人的标记**始终视为"错误"** |

> **说明**：定位模块离线意味着裁判系统无法获得该机器人的真实位置，因此采取保守判定策略——对被标记方有利（对方标记视为准确以产生易伤），对标记方不利（己方标记视为错误无法产生效果）。

### 11.3 激光检测模块离线

> 来源：④比赛规则手册 第5.1.5.5节

当空中机器人的**激光检测模块离线**时：

- 该空中机器人**始终视为被对方激光发射装置持续照射**
- 即被反制进度 P 值会持续增加，直至反制成功

> **说明**：这是一种惩罚性判定，防止空中机器人通过故意使激光检测模块离线来逃避雷达反制。

### 11.4 红牌罚下雷达

> 来源：④比赛规则手册 红牌罚下相关规定

当雷达被红牌罚下时：

- **激光发射装置及执行机构断电**：激光反制功能失效
- **多机通信断开**（仅能接收不能发送）：雷达丧失向己方队友发送信息的能力

### 11.5 犯规条款

| 条款 | 内容 |
|------|------|
| **R15** | 除雷达外，不得使用官方设备电源给自备设备供电 |
| **R22** | 三分钟准备阶段结束后，雷达遥控器需放置在指定区域 |
| **R56** | 不可主动攻击对方雷达（雷达受保护） |

## 12. 雷达竞技评分标准

> 来源：①参赛手册 雷达评分标准

雷达被纳入**机器人竞技奖**的评选范围，雷达是其中一个独立评选类别。

### 评分公式

```
时均雷达表现分数 = (累计准确标记时间 × 1 + 反制无人机次数 × 20 + 成功解析信息波难度等级 × 200) / 雷达所属队伍正式比赛总时长
```

### 评分要素分解

| 评分要素 | 权重系数 | 说明 |
|----------|---------|------|
| 累计准确标记时间 | × 1 | 雷达准确标记敌方机器人的总秒数 |
| 反制无人机次数 | × 20 | 成功反制对方空中机器人的次数 |
| 成功解析信息波难度等级 | × 200 | 成功解析的信息波最高难度等级 |

**分母**：雷达所属队伍的**正式比赛总时长**（秒）

> **分析**：
> - 从权重可以看出，**解析信息波**（×200）的单次收益最高，是雷达竞技评分中最有价值的能力
> - **反制无人机**（×20）次之
> - **准确标记**（×1）虽然单位权重最低，但由于可以大量累积时间，总量上仍有显著贡献
> - 评分是**时均值**，比赛时长越长分母越大，因此需要在整场比赛中保持稳定表现
